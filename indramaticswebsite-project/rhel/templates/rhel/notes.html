    • Manage files from the command line → 59Min
      • What is a filesystem?
        • It’s a system created by an OS to mange files. The system controls how data is saved or retrieved.
        • ext3, ext4, xfs are Linux file systems
        • NTFS, FAT are Windows file systems
      • Directory Listing Attributes
        • d → Directory
        • l (L) → Link
      • File Maintenance Commands
        • chgrp → Change the ownership of the file at group level
        • chown → Change the ownership of the file at user level
          • Can also be used for changing user and group ownership with use of colon : in between user and group name.
        • rmdir → Remove empty directory
      • Soft Links and Hard Links
        • inode: Pointer or number of a file on the hard disk
          • To get the inode of the file, run ls -li , i→ inode
        • Soft Link = Link will be removed if file removed or renamed
          • Soft linked files have different inodes.
          • Very similar as Windows Shortcuts
          • Also called as “Symlink” or “Symbolic Link”
          • Soft Links/symlinks/symbolic links can be created for files as well as directories.
          • Soft linking is allowed across partitions
        • Hard Link = Deleting, renaming, or moving the original file will not affect the hard link.
          • Hard linked files have same inodes.
          • Hard links are not allowed on Directories
          • Hard links only work within the same partition.
            • You get an error “Invalid cross-device link” if the directory where you are trying to create a hard link and the directory where the original file is located.
      • Input and Output Redirectors
        • 3 Redirectors in Linux
          • Standard Input (stdin) → File descriptor number = 0
          • Standard Output (stdout) → File descriptor number = 1
          • Standard Error (stderr) → File descriptor number = 2
        • Output (stdout) – 1
          • By default when running a command its output goes to the terminal
          • The output of a command can be routed to a file using > (new/overwrite) or >> (append) symbols.

        • Input (stdin) – 0
          • Input is used when feeding file contents to a file
          • E.g. cat < some_file
            • mail -s “office memo” allusers@abc.com < memoletter
        • Error (stderr) – 2
          • When command is executed via keyboard, that is stdin – 0
          • When output goes on the screen , that output is stdout – 1
          • When command produces any errors on the screen then it’s stderr – 2
            • We can use redirects to route errors from the screen
            • E.g. ls -l /non_existant_file 2> errorfile
              • telnet localhost 2> telnet_errors.log
      • Pipe (|)
        • A pipe is used by the shell to connect the output of one command to another command
        • ls -l | more → One page at a time. Space to go to the next page.
          • Note: If more is used with pipes, can not go back
          • Otherwise for files, when opened with more, you can go back by b

    • Manage local users and groups → 37Min
      • Commands
        • useradd
          • useradd -g superheros -s/bin/bash -c "user description" -m -d /home/spiderman spiderman
          • -g = group of the user
          • -s = default shell of the user
          • -c = user description / full name
          • -m = create user’s home directory
          • -d = home directory of the user
          • -M = No home directory (somewhat tricky when user logs in, refer here)
        • userdel
          • userdel -r spiderman
            • -r = delete the home directory as well
        • groupadd
          • groupadd superheros - create a new group
          • cat /etc/group – to verify
          • groupdel superheros - delete the group
        • groupdel
        • usermod
          • modify the group of the user (-G new_group username)
            • usermod -G superheros spiderman
            • The group of username spiderman which is spiderman by default gets a parent group called superheros
            • If we indeed want to change the group of file/directory from spiderman to superheros , we need use chgrp
          • chgrp (change the group of file)
            • chgrp -R superheros directory_name
            • -R = change every file under directory_name directory to superheros

      • Files
        • /etc/passwd
          • name:password:userid:groupid:description:homedirectory:shell
        • /etc/group
          • groupname:grouppassword:groupid:groupmembers
        • /etc/shadow
          • username:encrypted_password:days_since_lastpassword_change_absolute_from_epoch_time:minimum_number_days_before_user_can_change_password:maximum_number_of_days_before_password_change:days_for_warning:number_of_days_after_password_expirs_that_account_disabled
          • For changing above defaults, use /etc/login.defs
          • For changing for a particular user, use chage
          • has encrypted password as a second entry, if no password then the password for the user is not yet created
          • password expiry details
          • password length details
      • Create password
        • passwd <username>
      • Password age
        • chage - to change the age of password
          • -d = Last password change, days since epoch time that password was last changed
          • -m = The minimum number of days required between password changes
          • -M=The maximum number of days the password is valid
          • -W=The number of days before the password expirty that the user is warned
          • -I=The number of days after password expirty that the user is active
          • -E=Days since epoch time that account is disabled – abosolute date specifigying when the login may no longer be used.
        • Not used to change any user account setting – for that you use usermodcommand. This is just for password aging policy for a specific user
        • For chaning the defaults of password aging policy, use /etc/login.defsfile.
        • File: /etc/login.defs (login defaults file)
          • PASS_MAX_DAYS
          • PASS_MIN_DAYS
          • PASS_MIN_LEN
          • PASS_WARN_AGE

      • Switch User (su)
        • su - user_name (gets into the user’s default directory)
        • su user_name (gets into the same directory)
        • su - = become root
      • Sudoers
        • File /etc/sudoers
        • wheel group can give that to ALL users
        • You can add per users as well in that file, below root
        • Command to a user in a wheel group
          • usermod -aG wheel <user_name>
        • command to verify: sudo dmidecode
          • can’t run this command without sude access
    • Control access to files → 24Min
      • File Permissions
        • Command: chmod
        • -rwxrwxrwx.
          • User, group, others
          • -u, -g, -o
          • -a = for everyone
        • Rmove the write permissions of the <group>
          • chmod g-w <file_name>
        • Remove read permissions for every user
          • chmod a-r <file_name>
        • Add the write permission for the group of the file
          • chmod g+w <file_name>
        • Directory must have xpermissions – for us to be able to the directory
      • File ownership
        • Directory level permissions are necessary for editing, creating, or deleting files in that directory
        • User level and group level
        • chown - changes the ownership of the file
          • chown <user_name> <file/directory>
        • chgrp - changes the group ownership of the file
          • chgrp <group_name> <file/directory>
    • Monitor and mange Linux processes → 34Min
      • Processes and Jobs terminology
        • Application = Service, a program that runs (e.g. Word in windows, NTP, Apache in Linux)
        • Script = Commands (useradd, cd, pwd, etc.), applications are run with it
        • Process = when Application is started, it generates process id
        • Daemon = something that continuously run in the backround (keeps listening to incoming or outgoing traffic)
        • Threads = A process can have a multiple thread
        • Job = created by a scheduler to run those applications/serivces
      • Manage Processes
        • df -h
          • df = report file system disk usage space
          • devtmpfs and tmpfs are swap usage
        • du
          • du -BM / | sort -nr | more
            • Get size (B) in Megabytes (M) of / directory, sort in reverse order
        • uptime –> load metric can be useful if your system is slow
        • top
          • First line same as uptime
          • Tasks details
          • CPU → if it’s 95%, something is wronng, look for process
          • Memory details
          • Swap details
          • Columns:
            • PID: Process ID
            • USER: The owner of the process
            • PR: Process Priority
            • NI: The nice value of the process
            • VIRT: Amout of virtual memory used by the process
            • RES: Amout of resident memory used by the process
            • SHR: Amout of shared memory used by the process
            • S: Status of the process ((sleeping, running, etc)
        • free → memory information
          • free -m (for usage in MB)
        • lsof → List of open files
          • Some process can have these files open even if they exit
        • tcpdump → network activity
          • tcpdump -i <interface_name>
          • What is coming, what is going out, etc.
          • Run ifconfig to find your interface
            • lo-> local
            • vir* → virtual interface
            • eth* → Look for this
        • netstat → Network Statistics, router, gateway
          • netstat -rnv
        • ps → Process Status, snapshot of processes
          • ps -ef | more
          • ps -ef | grep <process_name>
        • kill
          • kill <process_id> → Kills gracefully
          • kill -9 <process_id> → Kills harshly, directly.
        • vmstat, iostat, ifstat, etc.
    • Control services and daemons → 20Min
      • Commands
        • systemctl --version → Check if systemd installed in your system
        • ps -ef | grep systemd → Check if systemd is running, first process has PID 1
        • systemctl --all → Check all running services
        • systemctl status|start|stop|restart application.service → To check the status, start, stop, and restart a service
          • systemctl status firewalld
          • systemctl start firewall
        • systemctl reload application.service → To reload the configuration of a service.
          • Reload is not going to impact the service, start/stop will impact the service
        • systemctl enable|disable application.service → To enable or disable a service at boot time
        • systemctl mask|unmask application.service → To enable or disable a service completely (at boot or manually)
          • Boot time will do the same excpet, if service B is requested when the boot is disabled for service A, it will start service A even if it’s disabled
          • If unmask then no matter what, service A will never start
      • service and process continuously in backround → daemons
      • services are controlled by systemctl → System Control
      • systemctl is a systemd utility that is responsible for controlling the systemd system and service manager
        • systemd is is a collection of system management daemons, utilities, and libraries which serves as replacement of System V init.d (init daemon)
        • systemd is a parent process of most of the daemons
      • Number of packages installed on the system
        • rpm -qa | wc -l → refer here for command description
    • Configure and secure SSH → 29Min
      • What is a Shell
        • Hardware → Kernel → Shell (bash, csh, ksh) → Utilities (cp, pwd, ls)
        • Open SSH is a package, runs by sshd service on port 22
      • SSH = Always encrypted
      • Following configurations for security
        • User timeout
          • Always copy the file of config file before editing
          • Idle Timeout Interval
          • Edit /etc/ssh/sshd_config file (with root)
            # uncomment (or add if not found) following lines
# timeout_interval = ClientAliveInterval * ClientAliveCountMax
    # except when ClientAliveCountMax is 0
    # in that case system will log you out right after first 
    #.   interval without any warning
ClientAliveInterval 600 # 10 minutes
ClientAliveCountMax 0 # send message how many times
          • systemctl restart sshd
        • Disable root login
          • /etc/ssh/sshd_config
          • Find PermitRootLogin to no
          • Restart the daemon
        • Disable Empty Passwords
          • PermitEmptyPasswords no → Uncomment and restart the daemon
        • Limit user’s SSH Access
          • /etc/ssh/sshd_config
          • Add following lines
          # allow following users
AllowUsers <user1> <user2>
        • Use different SSH Port
          • Edit /etc/ssh/sshd_config
          • Find Port 22
            • Make sure the port you are going to use is not used by anything
            • Uncommend and put in the port number that you need
        • SSH Keys
          • Configure SSH Keys
            • Generate keys on client machine (don’t if already have)
              • Check ~/.ssh/ files
              • If id_rsa.pub is present, just copy
            • Generate keys
              • ssh-keygen
            • Copy the Key to the server
              • ssh-copy-id root@remote_server_ip
            • The keys are copied to servers’s ~/.ssh/authorized_keys
            • Login and check
      • $ -> normal user
      • # -> root user

    • Manage networking → 33Min
      • Static vs Dynamic IP
        • Commands
          • ifconfig (interface configuration) → ip addr (newer)
            • Three interfaces
              • enps03 → for external communication
              • lo → Always same IP, processes inside the computer they use this interface to communicate with each other (loopback device)
              • virtualbridge → Virtual adapter
        • IP → Internet Protocol
        • Private IP → Local network (internal communication)
        • Public IP → Global IP (external communication)
        • Static IP → IP that does not change with reboot, manually assigned
        • DHCP → Dynamic Host Control Protocol
      • OS network components (Network interfaces, subnet mask, gateway, MAC address, etc.)
        • Network Interface
          • In physical computers, the ethernet ports connected to network cards gives you network interface
          • ifconfig→ look for inet in
        • MAC address
          • Hard coded that never chagnes → assigned to interface
          • ifconfig → look for ether in interface for external communication (enp0s3)
          • Assigned by manufacturer when the ethernet card is built
        • Subnet Mask
          • look for netmask in ifconfig output, it is your subnetmask
            • Allows/control the traffic
          • A set of IPs
        • Gateway
          • netstat -rnv
            • Anywhere you want to go outside your computer, you must go through the gateway
          • An IP assigned to your IP, allows to take your traffic to outside the local network
        • DNS (Domain Name System)
          • nslookup www.google.com
            • Look up the IP of a host
            • Also can see your DNS server here (Server)
            • In local environment, local gateway is usually DNS server
          • IP ↔ Hostname relationships

      • Compress and un-Compress Files
        • tar → Put all files in a bundle
          • Tar:
            • tar cvf <tarfile.tar> <directory_to_tar>
              • tar cvf current_dir .
                • Will give an error as you’re going to tar the dir and you’re in it
          • Untar:
            • tar xvf <tarfile.tar> -> Uncompress in the current directory

        • gzip → Gunzip = Compress the file
          • gzip <tar_file_to_compress>
            • Outputfile comes with .gz suffix
        • gzip -d → Uncompress files
          • gzip -d <gz file to uncompress>

        • Protocol for transferring files from Server A to Server B
        • Default port is Port 21
        • Watch this Udemy lecture for setting up FTP between server and client
      • SCP → Secure Transfer Protocol

    • Install and update software → 37Min
      • Package Management (rpm and yum)
        • yum / dnf commands. yum is deprecated, dnf is way to go in future for CentOS and RHEL
        • apt-get is used for other linux systems
        • YUM ( go online with this URL (in config files) and install a package to the system)

          • When already have a package on the system – when don’t have internet access
          • Commands
            • rpm -qa # query (q) all (a) packages installed on the system
            • rpm -ihv /tmp/package.rpm → to install package present at /tmp/package.rpm
        • To remove / uninstall package
          • Commands
            • rpm -e <package_name>
            • yum remove <package_name>
          • Both commands do same thing – remove packages

        • Major Version ≠ can not never use YUM command
        • Minior version:
          • yum update -y # dont ask questions, just update everything
        • Upgrade vs. Update
          • Upgrade: Deletes Packages (deletes old v5.0 package and installs new one)
          • Update: Preservers old packages (in case need to go back)
